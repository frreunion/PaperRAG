# 论文 RAG 助手需求文档

## 1. 项目背景

科研人员在阅读和管理论文时，通常会遇到以下问题：

- 论文数量多，难以快速定位关键信息。
- PDF 内容长，公式、图表、引用关系复杂。
- 普通大模型容易产生幻觉，回答缺少原文依据。
- 多篇论文之间的对比、归纳、综述整理耗时。
- 个人知识库无法和 AI 问答、检索、笔记流程打通。

因此，需要构建一个面向论文场景的 RAG 助手，支持论文上传、解析、索引、检索、问答、总结、对比和引用溯源。

## 2. 产品目标

构建一个支持论文上传、解析、检索、问答和综述分析的 AI 助手。

核心目标：

- 用户可以上传 PDF 论文，系统自动解析正文、标题、作者、摘要、章节、参考文献等信息。
- 用户可以基于单篇或多篇论文进行问答。
- 回答必须给出引用来源，包括论文名、页码、章节或原文片段。
- 系统支持论文摘要、方法梳理、创新点提取、实验结果分析、论文对比。
- 系统应尽量降低幻觉，无法从论文中找到依据时明确说明。

## 3. 目标用户

| 用户类型 | 需求 |
| --- | --- |
| 研究生 | 快速阅读论文、整理文献综述、理解方法和实验 |
| 科研人员 | 管理大量论文、跨论文对比、追踪技术路线 |
| 工程研发人员 | 从论文中提取可实现方案、复现实验细节 |
| 教师/导师 | 检查学生阅读情况、快速评估论文价值 |

## 4. 核心使用场景

### 4.1 单篇论文精读

用户上传一篇论文后，系统自动生成：

- 论文基本信息
- 摘要总结
- 研究问题
- 方法框架
- 主要贡献
- 实验设置
- 结果结论
- 局限性

用户可以继续追问：

- 这篇论文的核心创新是什么？
- 第三节的方法具体怎么实现？
- 作者的实验数据能否支撑结论？
- 这篇论文有哪些不足？

### 4.2 多篇论文对比

用户选择多篇论文后，可以询问：

- 这几篇论文的方法有什么区别？
- 哪篇论文更适合用于低资源场景？
- 请按研究问题、方法、数据集、指标、结论做一个对比表。
- 这些论文的发展脉络是什么？

### 4.3 文献综述辅助

用户上传某一方向的多篇论文后，系统帮助生成：

- 研究主题聚类
- 技术路线总结
- 时间线
- 代表性方法
- 存在问题
- 未来研究方向
- 初步综述大纲

### 4.4 论文复现辅助

用户可以询问：

- 这篇论文需要哪些数据集？
- 模型结构是什么？
- 训练参数有哪些？
- 复现难点在哪里？
- 代码实现时需要注意什么？

系统需要从论文中提取实现相关信息，并指出论文中缺失的信息。

## 5. 功能需求

### 5.1 用户与项目管理

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 用户注册/登录 | 支持基础账号系统 | P1 |
| 论文库管理 | 用户可以创建个人论文库 | P0 |
| 项目/课题文件夹 | 按研究方向管理论文 | P1 |
| 标签管理 | 给论文打标签，如 RAG、Agent、CV、通信 | P1 |
| 收藏/归档 | 收藏重要论文，归档旧论文 | P2 |

### 5.2 论文上传与解析

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| PDF 上传 | 支持单个或批量 PDF 上传 | P0 |
| 基本文档解析 | 提取正文文本、页码、段落 | P0 |
| 元信息提取 | 标题、作者、机构、年份、会议/期刊 | P0 |
| 章节识别 | 识别 Abstract、Introduction、Method、Experiment、Conclusion 等 | P0 |
| 参考文献提取 | 提取参考文献列表 | P1 |
| 图表说明提取 | 提取 figure/table caption | P1 |
| 公式识别 | 基础保留公式文本或 LaTeX | P2 |
| OCR | 支持扫描版 PDF | P2 |

### 5.3 向量索引与检索

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 文本切块 | 按章节、段落、token 长度切分 | P0 |
| Embedding | 将文本块转换为向量 | P0 |
| 向量数据库 | 支持 Chroma、FAISS、Milvus、pgvector 等 | P0 |
| Metadata 过滤 | 按论文、作者、年份、章节、标签过滤 | P0 |
| 混合检索 | 向量检索 + 关键词检索 BM25 | P1 |
| Rerank | 对召回片段重新排序 | P1 |
| 引用定位 | 保存页码、章节、chunk 来源 | P0 |

### 5.4 问答功能

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 单篇论文问答 | 基于某一篇论文回答问题 | P0 |
| 多篇论文问答 | 跨多篇论文检索并回答 | P0 |
| 引用溯源 | 回答中标注来源论文、页码、原文片段 | P0 |
| 无依据拒答 | 找不到依据时说明无法从论文中确认 | P0 |
| 追问上下文 | 支持连续对话 | P0 |
| 答案结构化 | 支持表格、列表、分点回答 | P0 |
| 原文高亮 | 点击引用跳转到 PDF 对应位置 | P1 |
| 中英文问答 | 支持中文问题、英文论文回答 | P0 |

### 5.5 论文分析功能

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 一键总结 | 生成论文摘要、贡献、方法、实验、结论 | P0 |
| 创新点提取 | 提取论文核心创新 | P0 |
| 方法流程图 | 生成方法步骤说明，后续可扩展为图示 | P1 |
| 实验分析 | 提取数据集、指标、baseline、结果 | P0 |
| 局限性分析 | 总结论文不足和未来工作 | P0 |
| 复现清单 | 提取模型、数据、参数、代码、硬件需求 | P1 |
| 论文评分 | 从创新性、严谨性、可复现性等维度评分 | P2 |

### 5.6 多论文对比功能

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| 多论文对比表 | 按问题、方法、数据集、指标、结论对比 | P0 |
| 技术路线总结 | 总结多篇论文之间的演进关系 | P1 |
| 研究主题聚类 | 自动聚类相似论文 | P2 |
| 文献综述大纲 | 基于选中论文生成综述结构 | P1 |
| 代表论文推荐 | 从论文库中找代表性论文 | P2 |

### 5.7 笔记与导出

| 功能 | 描述 | 优先级 |
| --- | --- | --- |
| AI 生成笔记 | 自动生成阅读笔记 | P1 |
| 用户手动笔记 | 用户可编辑和保存笔记 | P1 |
| Markdown 导出 | 导出总结、问答、综述 | P1 |
| BibTeX 导出 | 导出引用信息 | P2 |
| Word/LaTeX 导出 | 导出综述初稿 | P2 |

## 6. 非功能需求

| 类型 | 要求 |
| --- | --- |
| 准确性 | 回答必须尽量基于检索到的论文内容 |
| 可追溯性 | 每个关键结论应附带来源 |
| 稳定性 | PDF 解析失败时应提示原因，不应导致系统崩溃 |
| 性能 | 单篇论文问答响应建议控制在 5-15 秒内 |
| 可扩展性 | 后续可接入更多模型、数据库、文档格式 |
| 安全性 | 用户论文数据隔离，避免跨用户泄露 |
| 成本控制 | 控制 embedding、rerank、LLM 调用频率 |
| 可观测性 | 记录检索片段、模型输入、输出、耗时、错误 |


## 8. RAG 流程设计

### 8.1 论文入库流程

```text
上传 PDF
  ↓
解析 PDF 文本、页码、章节
  ↓
提取元信息
  ↓
文本清洗
  ↓
按章节和语义切块
  ↓
生成 embedding
  ↓
写入向量数据库
  ↓
写入论文元数据
  ↓
完成入库
```

### 8.2 问答流程

```text
用户提问
  ↓
问题改写/意图识别
  ↓
选择检索范围：单篇/多篇/全部论文库
  ↓
向量检索 + 关键词检索
  ↓
rerank 重排序
  ↓
构造上下文
  ↓
调用 LLM 生成回答
  ↓
引用来源绑定
  ↓
返回答案 + 来源片段
```

## 10. MVP 版本范围

### 10.1 MVP 必须实现

- PDF 上传
- 论文文本解析
- 论文元信息提取
- 文本切块
- embedding 入库
- 单篇论文问答
- 多篇论文问答
- 回答引用来源
- 一键论文总结
- 多论文对比表
- 基础聊天记录
- 简单前端界面

### 10.2 MVP 暂不实现

- OCR
- 复杂公式解析
- 自动生成可视化流程图
- Word/LaTeX 导出
- 多用户权限系统
- 论文推荐系统
- 浏览器插件
- 复杂 agent 多步规划

## 11. 验收标准

| 模块 | 验收标准 |
| --- | --- |
| PDF 上传 | 能成功上传并解析常见 arXiv PDF |
| 文本解析 | 能保留页码和章节信息 |
| 检索 | 用户问题能召回相关论文片段 |
| 问答 | 回答与论文内容一致，不编造明显事实 |
| 引用 | 每个关键回答至少给出 1 个来源 |
| 总结 | 能输出结构化论文阅读笔记 |
| 对比 | 能对 2-5 篇论文生成合理对比表 |
| 失败处理 | 解析失败、无检索结果时有明确提示 |
| 性能 | 单篇论文问答在可接受时间内返回 |

## 12. 评测指标

| 指标 | 说明 |
| --- | --- |
| Recall@K | 正确片段是否被检索出来 |
| Citation Accuracy | 引用是否真的支持回答 |
| Faithfulness | 回答是否忠于原文 |
| Answer Relevance | 回答是否切题 |
| Latency | 响应耗时 |
| Cost per Query | 单次问答成本 |
| User Rating | 用户对答案满意度 |

## 13. 学习与实现优先级

推荐开发顺序：

1. PDF 解析 + chunking
2. embedding + 向量检索
3. 基础 RAG 问答
4. 引用溯源
5. 论文总结模板
6. 多论文对比
7. 检索效果优化：BM25、rerank、query rewrite
8. 评测集和自动评测
9. 前端体验优化
10. 部署、权限、成本控制
